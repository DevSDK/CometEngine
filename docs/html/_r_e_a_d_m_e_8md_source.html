53  ## Design of Librarys
54 
55  난수를 생성 라이브러리를 자체 제공한다. ex) 메르센 트위스터
56 
57  파일을 zip형태로 압축하거나, 리소스를 직접 컴파일함[암호화 지원]
58 
59 • 잘 알려진 파일 형식 ex) PNG, JPEG, BMP 등등은 에디터에 불러와 파싱
60 
61 • 자체적인 에니메이션 메커니즘이 필요 (리깅, 스켈레톤 등)
62 
63  -만약 필요할경우, Blender 같은 Util에서 만드는 데이터를 파싱할 필요 있음.
64 
65 • 네트워크의 경우 Serializer를 제공, Bit 단위 전송을 최대한 지원.
66 
67  -C# 자체 통신은 사용하지 않고 C++ WinSock을 사용할 것, 다만 Wrapping에 주의
68 
69 • 게임게발자가 로우레벨 시스템을 고려하지 않도록 설계하면서도 언제든지 로우레벨에 접근이 가능하도록 설계
70 
71 • In-Game Profiler을 개발해 CPU,GPU, Memeory, Allocation등등을 수집 및 출력
72 
73 • 필요할 경우 구조를 State Machine + Pipe Line로 모듈을 구성
74 
75  - ex) Animation Pipeline, Resource Processing PipeLine, etc..

157 ## Design Of Librarys
158 
159 •Provide Random Number Library. ex) Mersenne Twister, or external library
160 
161 •Compress Resources like zip file or Compile Resource. [Support Encryption]
162 
163  • Load to Editor and parsing data Well-Known File Format ex) PNG, JPEG, BMP and ETC
164 
165  • Probably Need Own Animation Method ( Rigging, Skeleton )
166 
167  - Probably Need Parsing Data from well-known utils ex) Blender.
168 
169  • Network Module support Serializer, and make sure send a bit data.
170 
171  - Do Not Use ".Net"and "C# Socket." Using C++ WinSock. and make sure Wrapping.
172  • Make sure the Game Developer focus on High-Level. But The CometEngine provide Low-Level System
173 
174  • Develop In-Game Profiler. For CPU,GPU, Memeory and Allocation Information Collect and Visualize
175 
176  • If I Need, module strcuture change to State Machine + Pipe Line
177 
178  - ex) Animation Pipeline, Resource Processing PipeLine, etc..
